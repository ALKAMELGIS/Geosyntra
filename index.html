Agro Cloud

تظهر هذه الصفحة عندما يكون مصدر GitHub Pages هو «Deploy from a branch» من جذر المستودع (/).

التطبيق الحقيقي يُبنى في frontend/dist ويُنشر عبر سير العمل Deploy to GitHub Pages فقط. لا تُلتزم مخرجات البناء في الفرع لأن GitHub قد يرفض الدفع عند وجود رموز داخل ملفات .js المجمّعة.

  1. Settings → Pages → Build and deployment — اختر المصدر GitHub Actions (وليس النشر من الفرع من الجذر).
  2. ادفع إلى main أو شغّل السير يدوياً من تبويب Actions.
  3. إذا علق النشر: Settings → Environments → github-pages.

فتح إعدادات Pages سير نشر Pages

بعد نجاح النشر عبر Actions، افتح التطبيق (HashRouter — الرابط يجب أن ينتهي بـ #/):

https://alkamelgis.github.io/AgroCloud/#/

ملاحظة: طالما النشر من الفرع فقط، الرابط أعلاه لن يحمّل حزم React لأنها غير موجودة في الجذر.

English: This page appears when GitHub Pages uses Deploy from a branch at repository / (root). The real app is built into frontend/dist and published only by the Deploy to GitHub Pages workflow. Do not commit frontend/dist or docs/ to git: push protection may block embedded tokens in bundles.
