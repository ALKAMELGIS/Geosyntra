# Usability Testing Plan (10+ users)

## Goal
Validate the redesigned UI against common workflows and measure satisfaction.

## Participants
- Minimum: 10 users.
- Mix of: admin users, data-entry users, GIS users.

## Test Tasks (example set)
1. Login and navigate to a dashboard.
2. Open GIS Map and add a layer (ArcGIS Feature Service).
3. Create a relationship and preview the generated form.
4. Edit a record in the generated form and save.
5. Navigate to Data Entry and submit a record.

## Metrics
- Task completion rate (per task)
- Time-on-task (optional)
- Error rate (user-reported + observed)
- Satisfaction score (target ≥ 90% satisfied)

## Satisfaction Survey (post-test)
Question (single choice):
- “Overall, how satisfied are you with the new interface?”
Options:
- Very satisfied
- Satisfied
- Neutral
- Dissatisfied
- Very dissatisfied

Scoring:
- Satisfaction % = (Very satisfied + Satisfied) / total responses * 100
- Target: ≥ 90%

## Session Script (moderator)
1. Intro: “We are testing the interface, not you.”
2. Ask participant role and prior experience.
3. Run tasks without coaching; allow clarifying questions only.
4. Note points of confusion and friction.
5. Post-test survey and short interview: “What was hardest?”, “What felt fastest?”, “What would you change?”

## Output
- Summary report with:
  - Satisfaction %, task completion rates
  - Top issues and severity
  - Recommended fixes and follow-up test plan
